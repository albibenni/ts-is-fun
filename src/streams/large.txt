#### GET /api/v1/users

**Description:** Retrieves a list of all users in the system with pagination support.

**Parameters:**
- page (integer, optional): Page number for pagination (default: 1)
- limit (integer, optional): Number of items per page (default: 50, max: 100)
- sort (string, optional): Sort field (options: name, email, created_date)
- order (string, optional): Sort order (options: asc, desc)

**Response Format:**
```json
{
  "status": "success",
  "data": {
    "users": [
      {
        "id": 12345,
        "name": "John Doe",
        "email": "john.doe@example.com",
        "created_date": "2024-01-15T10:30:00Z",
        "status": "active"
      }
    ],
    "pagination": {
      "current_page": 1,
      "total_pages": 25,
      "total_items": 1234,
      "items_per_page": 50
    }
  }
}
```

#### POST /api/v1/users

**Description:** Creates a new user account in the system.

**Request Body:**
```json
{
  "name": "string (required, 2-100 characters)",
  "email": "string (required, valid email format)",
  "password": "string (required, minimum 8 characters)",
  "role": "string (optional, default: user)"
}
```

**Response Codes:**
- 201: User created successfully
- 400: Invalid request data
- 409: Email already exists
- 500: Internal server error
